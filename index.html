Before You Start -- Quick FAQ

How should I think about Claude?
Think of it as a very capable new hire who needs clear instructions but has no context on how we do things. The more specific you are about what you want, the better the output. You're managing it the same way you'd manage a person -- clear scope, clear expectations, clear feedback.
Is my data safe?
On the Team plan, Anthropic does not train on your data. Your conversations and uploaded documents stay private to our team workspace. Claude processes your input to generate a response and does not retain it afterward.
What if Claude gets something wrong?
It will, sometimes. That's expected. Always review Claude's output before using it -- think of it as a very fast first draft, not a finished product. If something's off, tell it what to fix in the same conversation and it'll adjust. A human check is always the last step.
Do I need to be technical?
Not at all. If you can type a message, you can use Claude. The prompt templates we'll set up this week are copy-paste -- you just fill in one input field with whatever data or notes you already have.
What about data privacy, licensing, and compliance?
Anthropic takes data protection seriously. On the API and Business/Enterprise plans, your data is never used for training. API inputs and outputs are retained for up to 30 days for trust & safety, then deleted. Anthropic holds SOC 2 Type II certification. Claude is also available through AWS Bedrock and Google Vertex AI if your data needs to stay within those environments. For enterprise-specific data processing agreements (DPAs), custom contractual requirements, or licensing questions, reach out to Anthropic's sales team directly at anthropic.com/contact-sales. Set Up Your 3 Projects
~30 minutes

What's a Project?

A Project is a workspace in Claude that remembers context between conversations. You upload documents (templates, past reports, guidelines) and set instructions so Claude already knows how we work -- no re-explaining every time. Think of it as giving that new hire a proper onboarding folder.

Create the 3 Projects

We've already identified the three deliverables that make sense to start with (see the reference table below for what to upload into each one).

Generate custom instructions for each Project

Open a chat inside each Project and paste this prompt. Claude will read what you uploaded and write tailored instructions for that deliverable.

I'm setting up this Claude Project for one specific recurring deliverable my team produces: [NAME -- e.g., "quarterly RE reporting"].

I've uploaded example outputs and background docs.

Your job: Generate a Project instruction set I can paste into this Project's instructions field. It should include:

WHAT THIS DELIVERABLE IS: One sentence describing the output
WHO IT'S FOR: The audience (internal, client, leadership, etc.)
TONE & FORMAT: How it should read, how long it should be, what structure it follows
QUALITY BAR: What separates a good version from a bad one -- based on the examples I uploaded
GUARDRAILS: What to never do in this specific deliverable (e.g., never speculate on timelines, always include next steps, never exceed one page)

Format the output as a ready-to-paste instruction block.
Why this matters: Without instructions, Claude gives generic output. With them, it produces work that matches our standards from the first message. This is the single biggest thing that separates useful AI from toy demos.

The idea

Nobody should have to stare at a blank chat wondering "What do I type?" Today you're creating a simple one-line copy-paste template for each Project, and then testing one to make sure it actually works.

Generate templates

Inside each of the 3 Projects, open a new chat and paste this prompt:

Based on the instructions in this Project, write me the shortest possible prompt template my teammates can copy-paste to produce this deliverable.

Rules:
1. One sentence max
2. ONE [INPUT] field (raw notes, a rough draft, or bullet points they already have)

The template should rely on the Project instructions for everything else -- tone, format, quality, guardrails. Don't repeat any of it in the template.
